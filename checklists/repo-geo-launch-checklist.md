# Repository GEO Launch Checklist

## Homepage

- Title matches a real search intent
- README answers the user's first question in the first screen
- One-sentence positioning is clear
- 2-4 problem-shaped headings exist
- Use cases are concrete

## Metadata

- Repository description is filled
- Topics are filled
- Homepage URL is set
- `CITATION.cff` exists
- `llms.txt` exists
- `AGENTS.md` exists

## Pages

- GitHub Pages is enabled
- At least 3 keyword landing pages exist
- `robots.txt` exists
- `sitemap.xml` exists
- canonical URLs are set

## Trust Signals

- License exists
- README links to docs
- README includes a visible star-history chart with the real `OWNER/REPO` slug
- docs link back to repo
- examples or templates exist

## Distribution

- Profile README links to the repo
- One external post links back with keyword anchor text
- One problem-shaped X thread exists
